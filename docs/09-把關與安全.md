# 09 把關與安全

AI 產出看起來專業，不代表內容正確。投影片中提醒兩種風險：漂亮但全錯，以及一味附和你。

## 教師端紅線

不要上傳：

- 學生姓名
- 學號
- 聯絡方式
- 成績
- 個別作業與評語
- 未匿名化問卷
- 未授權公開分享的教材、書籍 PDF、學生作品

如果一定要分析學生資料，先匿名化，並遵守學校政策與所在地法規。

## 重要產出要人工檢查

| 產出 | 檢查方式 |
|---|---|
| 考卷解答 | 抽樣重算；重要題目全部重算 |
| 數學反例 | 自己驗算定義與條件 |
| 文獻引用 | 回原文確認，不信任 AI 生成 citation |
| 學生回饋 | AI 只草擬，教師最後判斷 |
| 課程公告 | 確認日期、地點、連結、政策 |

## 問法要改

不要問：

```text
這樣寫對嗎？
```

改問：

```text
請扮演嚴格審稿人，找出這份內容的錯誤。
請特別檢查：
1. 事實或數學是否錯誤
2. 是否有過度宣稱
3. 是否有學生會誤解的地方
4. 哪些內容需要我人工查證
```

## 獨立驗證

同一個 AI 產生的內容，不應該是唯一驗算者。可以用三種方式：

1. 自己人工檢查。
2. 開新對話，貼上目標與內容，請它挑錯。
3. 對主 agent 說：「如果需要獨立審查，請你自己安排另一個視角，最後只整合重點給我。」

這裡的獨立審查不代表老師要自己學會 subagent dispatch。老師要做的是要求「需要另一個視角挑錯」；主 agent 再決定怎麼處理。

## 發布前檢查清單

- [ ] 沒有學生個資。
- [ ] 沒有未授權公開素材。
- [ ] 所有連結可開。
- [ ] 教材中的日期、地點、考試範圍已確認。
- [ ] 答案、公式、反例已人工抽查。
- [ ] README 有說明 AI 內容需要教師最後判斷。
