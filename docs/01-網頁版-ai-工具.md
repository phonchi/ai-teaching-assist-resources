# 01 網頁版 AI 工具

網頁版 AI 工具適合做「說」的工作：問答、改寫、摘要、產生範例、草擬教案。對第一次接觸 AI 教學工具的老師來說，這是第一階，不是跳過的暖身。等你能把需求說清楚，再進到桌面版 agent 讀檔、改檔、跑檢查。

## 工具入口

| 工具 | 入口 | 適合用途 |
|---|---|---|
| ChatGPT | <https://chatgpt.com/> | 快速問答、改寫、圖片/語音功能、一般助理 |
| Claude.ai | <https://claude.ai/> | 長文閱讀、教材草稿、精緻寫作、規格化交辦 |
| Gemini | <https://gemini.google.com/> | Google 生態、長文件、Gmail/Docs 相關情境 |
| Perplexity | <https://www.perplexity.ai/> | 需要網路搜尋與引用來源的問答 |

## 什麼時候用網頁版？

- 你只需要文字草稿。
- 你不想安裝任何東西。
- 你要先把想法問清楚，再整理成桌面版 agent 的單次任務規格。
- 你要讓 AI 幫你整理教案、活動、討論題或常見錯誤。

## 基本問法

不要只寫：

```text
幫我做一份教案。
```

改成：

```text
你是一位大一微積分助教。
我要上一堂 50 分鐘的課，主題是多變數函數的偏導數。
學生已學過單變數微分，但還不熟 3D 圖像。

請產出：
1. 3 個學習目標
2. 50 分鐘課程流程
3. 2 個課堂提問
4. 1 題課後練習

限制：使用繁體中文；不要超出偏導數與幾何直觀；動工前先反問我需要確認的地方。
```

## 常見錯誤

- 問太大：例如「幫我設計整門課」。
- 沒給學生程度：AI 會自己猜。
- 沒給驗收標準：不知道怎樣算完成。
- 只問「對不對」：AI 容易順著你說。

## 建議習慣

每個任務結束後，不要只收下答案。先請 AI 回顧你的交辦方式，記下常見模糊點。等你開始用 Codex app 或 Claude Code desktop app 處理教材資料夾，再把反覆出現的規矩寫進 `AGENTS.md` 或 `CLAUDE.md`。完整做法放在 [回顧用法與 insights](05-工作守則與交辦模板.md#回顧用法與-insights)。
