# 11 Subagent 怎麼用

Subagent 可以想成「分身助教」。主對話負責決策與整合，subagent 負責去做一個邊界清楚的調查、搜尋、審查或對照工作，最後只把結論帶回來。

> 進階提醒：Subagent 適合大量搜尋、獨立審查、多份教材對照。第一次工作坊先學會網頁版 AI、GitHub Desktop、桌面版 agent 的只讀與小修改流程；不要一開始就派多個助理。

投影片中的比喻是：不要把整本書都搬到你的桌上，請助理去讀，只帶一頁結論回來。

## 什麼時候適合派 subagent？

適合：

- 大範圍搜尋：找出某個概念散在哪些檔案。
- 獨立審查：請另一個視角挑錯。
- 對照檢查：比較 A/B 版考卷、講義與解答是否一致。
- 平行工作：多份教材各自檢查，最後整合結果。

不適合：

- 需要你即時互動討論的設計決策。
- 任務很小，自己 5 分鐘內能看完。
- 任務依賴主對話裡很多沒寫清楚的背景。

## Prompt 要自我完備

不要寫：

```text
根據剛剛講的去查一下。
```

要寫：

```text
請用獨立審查者角度檢查 docs/02-notebooklm.md。

背景：這份文件是給第一次使用 NotebookLM 的教師。
目標：找出新手會卡住、步驟不清楚、或安全提醒不足的地方。
範圍：只看這個檔案，不要修改檔案。
輸出：列出最多 8 個問題，每個問題附一個可直接改寫的建議。
```

## 常見 subagent 類型

不同工具支援的名稱不一定一樣，但常見角色如下：

| 類型 | 適合任務 |
|---|---|
| Explore | 大範圍 codebase 或資料夾搜尋 |
| Plan | 複雜實作策略與 trade-off |
| general-purpose | 開放式調查、整理資料、找候選方案 |
| code-reviewer | 程式碼與文件變更審查 |
| security-auditor | 權限、個資、token、資料安全檢查 |
| test-engineer | 測試策略與驗收檢查 |

## 教師工作流範例

### 範例 1：檢查考卷

```text
請用獨立審查者角度檢查這份小考題目與解答。

背景：課程是大一微積分，範圍是偏導數與多重積分。
目標：找出數學錯誤、題目不清楚、解答跳步、A/B 版難度不一致之處。
限制：不要改檔案，只回報問題。
輸出：依嚴重程度排序，附建議修正方向。
```

### 範例 2：檢查 README

```text
請檢查這個 GitHub README 是否適合第一次接觸 AI 教學工具的學員。

請特別看：
1. 第一屏是否知道要去哪裡點
2. 安裝步驟是否分平台
3. 是否有個資與版權提醒
4. 是否有過期或不可信連結
```

### 範例 3：平行查多份講義

如果有多份講義，可以把每份講義交給不同 subagent，各自回答：

```text
請閱讀第 3 章講義，只整理這份講義中和「鏈鎖律」相關的段落。
輸出：概念摘要、頁碼或章節位置、學生可能誤解之處。
```

最後主對話再整合所有 subagent 的結論。

## 跟 Skill、MCP 的差別

| 概念 | 解決什麼 |
|---|---|
| Subagent | 把一個任務交給獨立 context 的助理去做 |
| Skill | 和 agent 合作，把跑順或反覆摩擦的流程整理成可重複使用的工作手冊 |
| MCP | 讓 AI 接上外部工具或資料來源 |

三者可以一起用：subagent 依照 Skill 的流程，透過 MCP 查資料，最後把結論回報主對話。
