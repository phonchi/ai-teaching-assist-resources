# 05 工作守則與交辦模板

投影片的核心觀念之一是：AI 不會消除模糊，它會放大模糊。最有效的改善方式，是把你的教學偏好與紅線寫成一份「工作守則」。

## 一頁教學工作守則

可以把這段放到 Claude、ChatGPT、Gemini、Claude Code、Codex CLI 或 Gemini CLI 的對話開頭：

```text
# 我的教學工作守則

## 我教的課
- 課程：微積分、資料結構
- 學生：大一到大二
- 語言：繁體中文

## 出題慣例
- 題目要明確標示章節與學習目標。
- 解答版要逐步寫出關鍵理由。
- 多版本考卷要保持題型對應，不只隨便換文字。

## 教材慣例
- 複雜概念優先用圖像、例子、反例說明。
- 數學式要使用標準 LaTeX 表示。
- 網頁教材要能在手機與電腦閱讀。

## 偏好
- 先講結論，再列步驟。
- 動工前先反問我最多 3 個關鍵問題。
- 如果要修改超過 3 個檔案，先列計畫。

## 紅線
- 不處理學生姓名、學號、成績、聯絡方式。
- 不公開未授權教材或學生作品。
- 關鍵答案、解答、反例都要提醒我人工檢查。
```

## 六要素交辦模板

```text
目標：
範圍：
格式：
限制：
範例：
驗收標準：

動工前，請先反問我需要確認的地方。
```

範例：

```text
目標：做一份 NotebookLM 入門教學。
範圍：只教建立 notebook、上傳講義、問答、產生學習指南。
格式：GitHub README，可直接複製貼上。
限制：繁體中文；不要假造功能；不要放學生個資。
範例：以一份微積分講義 PDF 為例。
驗收標準：新手 10 分鐘內能建立第一個 notebook 並問出有引用的答案。

動工前，請先反問我需要確認的地方。
```

## 讓 AI 先反問你

如果你懶得寫完整規格，至少加這句：

```text
請不要直接動工。請先列出最多 3 個你需要我確認的問題，並提供選項。
```

這能降低 AI 自己腦補的機率。

## 把關 prompt

不要問：

```text
這樣對嗎？
```

改問：

```text
請用挑錯的角度檢查這份內容。
請列出：
1. 可能錯誤的數學或事實
2. 可能誤導學生的說法
3. 需要人工查證的地方
4. 建議我如何修改
```

## 回顧用法與 insights

投影片中的 `/insights` 對應「精進迴圈」：不是檢查這一次答案對不對，而是回顧你怎麼交辦、哪裡反覆卡住、哪些流程可以固定下來。支援 `/insights` 的工具可以直接輸入：

```text
/insights
```

如果你使用的工具沒有這個指令，就用等價 prompt：

```text
請回顧我這次的交辦方式。
請列出：
1. 我哪些地方交代得清楚，值得保留？
2. 哪些目標、範圍、格式、限制或驗收標準太模糊？
3. 你剛剛在哪些地方需要猜？
4. 哪些慣例應該寫進我的工作守則？
5. 哪些重複流程下次可以封裝成 Skill 或固定模板？
```

回顧後不要全部照單全收，只做三件事：

1. 把真正會重複出現的偏好寫回「一頁教學工作守則」。
2. 把常常漏寫的條件補進「六要素交辦模板」。
3. 若某個流程已經重複三次以上，再考慮升級成 [Skill](06-skills.md)。
