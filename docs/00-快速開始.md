# 00 快速開始

這一頁給第一次參加工作坊的學員。目標不是一次學會所有工具，而是在 30 分鐘內完成四件事：

1. 建立一個安全的教材工作區。
2. 把長期規矩寫進 agent 會讀的指令檔。
3. 建立一個教材修改的存檔點。
4. 讓 agent 先觀察、再計畫、再動手。

## A. 先準備帳號

| 帳號 | 用途 | 入口 |
|---|---|---|
| Google | NotebookLM、Gemini、Google Drive | <https://accounts.google.com/> |
| GitHub | 教材版本歷史、協作、課後資源 | <https://github.com/> |
| OpenAI | ChatGPT、Codex CLI | <https://chatgpt.com/> |
| Anthropic | Claude.ai、Claude Code | <https://claude.ai/> |

只想做零門檻練習的人，先有 Google 帳號做 NotebookLM，再用 GitHub Desktop 建立 Git 存檔點即可；GitHub 帳號是雲端協作與課後資源頁的延伸。

## B. 建立 agent 工作區十分鐘練習

先建立測試資料夾，不要直接拿正式教材練習。

```bash
mkdir teaching-agent-test
cd teaching-agent-test
git init
```

接著建立工作區指令檔。用 Claude Code 時建立 `CLAUDE.md`；用 Codex 或其他遵守 AGENTS 慣例的 agent 時建立 `AGENTS.md`。

```text
# 我的教學工作守則

## 課程與語言
- 課程：請改成你的課名
- 語言：繁體中文

## 工作方式
- 修改前先說明會讀哪些檔案、會改哪些檔案。
- 動工前先反問我最多 3 個關鍵問題。
- 修改後請摘要 git diff，列出需要老師人工確認的地方。

## 紅線
- 不處理學生姓名、學號、成績、聯絡方式。
- 不公開未授權教材或學生作品。
- 不要自行 push 或發布，除非我明確同意。
```

把教材副本放進資料夾後，先存第一版：

```bash
git add .
git commit -m "Save original teaching material"
```

## C. NotebookLM 五分鐘練習

1. 打開 <https://notebooklm.google.com/>。
2. 建立一個新的 notebook。
3. 上傳一份可以分享的講義、簡報或 PDF。
4. 問：

```text
請用繁體中文整理這份教材的 5 個重點。
每一點都要附上來源引用。
```

5. 再問：

```text
請根據這份教材產生 5 題練習題。
不要使用教材以外的知識。
```

## D. Agent 小任務十分鐘練習

先不要叫 agent 改正式教材。在剛剛的測試資料夾中啟動 agent，然後交辦：

```text
請閱讀這份教材。
先不要修改檔案。
請整理：
1. 這份教材的目標
2. 學生可能卡住的地方
3. 你建議修改的 3 個位置
4. 如果要修改，你會改哪些檔案
```

這一步的重點是讓 agent 先「觀察」，不要直接動手。

## E. Git 存檔點檢查

如果前面還沒有 commit，現在補上：

```bash
git init
git add .
git commit -m "Save original teaching material"
```

如果你不熟 terminal，可以用 GitHub Desktop：

1. 安裝 [GitHub Desktop](https://desktop.github.com/)。
2. 建立一個 local repository。
3. 把教材副本放進去。
4. 按 `Commit` 建立第一個存檔點。

這個存檔點的意思是：接下來你可以讓 agent 改副本，改壞了還能回來。

## F. GitHub README 延伸練習

1. 到 <https://github.com/new> 建立新 repo。
2. 勾選 `Add a README file`。
3. 在 README 寫三段：
   - 這是什麼課。
   - 學員應該先看哪份資料。
   - 有問題要去哪裡問。
4. 按 `Commit changes`。

這就是最小的課後資源頁。之後可以慢慢加入講義、連結、FAQ。

## G. 單次任務交辦模板

長期規矩已經放在 `CLAUDE.md` 或 `AGENTS.md`。每次對 agent 交辦時，只需要補本次任務的規格：

```text
我正在準備一份教學資源頁。

目標：讓學生能在課後找到工具連結與第一步教學。
範圍：只包含本次課程提到的工具，不展開無關資源。
格式：請用 README.md 的結構回答。
限制：使用繁體中文；不要放學生個資；不要假造連結。
驗收標準：讀者 3 分鐘內能找到該下載或註冊哪個工具。

動工前，請先反問我最多 3 個需要確認的問題。
```

## H. 完成後要檢查

- 連結能不能打開。
- 是否不小心放入學生個資。
- Agent 是否產生了不存在的網址或功能。
- Git 是否已有修改前的存檔點。
- Agent 是否有先列計畫、再動手、再回報需要人工確認的地方。
- 如果這次合作有摩擦，是否已請 agent 用 `/insights` 或回顧 prompt 整理，下次要寫回 `CLAUDE.md` / `AGENTS.md` 的規矩。
