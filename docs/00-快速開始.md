# 00 快速開始

這一頁給第一次參加工作坊、平常只用過網頁版 AI 的學員。目標不是一次學會所有工具，而是在 30 分鐘內完成四件事：

1. 用網頁版 AI 或 NotebookLM 完成一個零風險教材整理。
2. 用 GitHub Desktop 建立教材副本與存檔點。
3. 用 Codex app 或 Claude Code desktop app 讓 agent 只讀教材、不要修改。
4. 確認安全流程後，再做第一個小修改。

## A. 先準備帳號

| 帳號 | 用途 | 入口 |
|---|---|---|
| Google | NotebookLM、Gemini、Google Drive | <https://accounts.google.com/> |
| GitHub | 教材版本歷史、協作、課後資源 | <https://github.com/> |
| OpenAI | ChatGPT、Codex app | <https://chatgpt.com/> |
| Anthropic | Claude.ai、Claude Code desktop app | <https://claude.ai/> |

只想做零門檻練習的人，先有 Google 帳號做 NotebookLM，再用 GitHub Desktop 建立 Git 存檔點即可。Codex app 與 Claude Code desktop app 是下一步，不是一開始就要會 terminal。

## B. 網頁版 AI 十分鐘暖身

先用你已經熟悉的網頁版工具練習，不碰本機檔案，也不讓 AI 直接改教材。

可選一個工具：

- ChatGPT：<https://chatgpt.com/>
- Claude.ai：<https://claude.ai/>
- NotebookLM：<https://notebooklm.google.com/>

拿一份可以分享的教材，問：

```text
請用繁體中文整理這份教材的 5 個重點。
請另外列出學生可能卡住的 3 個地方。
```

再問：

```text
請根據這份教材產生 5 題練習題。
不要使用教材以外的知識。
請標出每題對應的教材段落或來源。
```

這一步只練習「把需求說清楚」。不要急著安裝 agent。

## C. GitHub Desktop 十分鐘：建立教材存檔點

Agent 會讀檔、改檔；所以在讓 agent 動手前，先建立教材副本與存檔點。

1. 安裝 [GitHub Desktop](https://desktop.github.com/)。
2. 建立一個 local repository，例如 `teaching-agent-test`。
3. 把一份不含學生個資、可以練習的教材副本放進去。
4. 在 GitHub Desktop 的 `Changes` 畫面確認新增檔案。
5. Commit message 寫：

```text
Save original teaching material
```

這個 commit 的意思是：接下來 agent 改壞副本，也能回到第一版。

## D. 桌面版 agent 十分鐘：只讀，不改

選一個低摩擦入口：

- Codex app：<https://developers.openai.com/codex/app>
- Claude Code desktop app：<https://code.claude.com/docs/en/overview>

在 app 中選剛剛的教材資料夾。第一次只交辦「讀取與回報」，不要改檔：

```text
請先只讀取這個資料夾，不要修改任何檔案，也不要執行會寫入的指令。

請用三段回報：
1. 這份教材看起來在教什麼
2. 學生可能卡在哪裡
3. 如果要改善，你會建議改哪些檔案與原因
```

這一步的重點是確認 agent 真的看得到正確資料夾，也知道先觀察、再計畫。

## E. 第一次安全修改

確認上一節回報合理後，才做一個很小的修改。例如只改 README 或一小段講義副本：

```text
請只修改 README.md。

目標：補上一段「這份教材適合誰」。
範圍：只新增 3-5 行，不要改其他段落。
限制：使用繁體中文；不要新增不存在的連結；不要碰學生資料。
驗收標準：第一次打開 README 的學生能知道自己該不該讀這份教材。

動工前，請先說明你會改哪個檔案與改動重點。
```

修改後用 GitHub Desktop 或 agent app 的 diff / review 畫面檢查：

- 是否只改了你允許的檔案。
- 是否有新增錯誤連結或不該公開的內容。
- 是否需要老師人工修正。

確認後再 commit。

## F. GitHub README 延伸練習

如果你想做課後資源頁，再到 <https://github.com/new> 建立新 repo。

1. 勾選 `Add a README file`。
2. 在 README 寫三段：
   - 這是什麼課。
   - 學員應該先看哪份資料。
   - 有問題要去哪裡問。
3. 按 `Commit changes`。

這就是最小的課後資源頁。之後可以慢慢加入講義、連結、FAQ。

## G. 第二輪才做：工作區指令檔

當你已經做過 1-2 次「只讀、列計畫、小修改、看 diff」後，再請主 agent 幫你草擬長期規矩：

- Codex app / Codex CLI：最後可存成 `AGENTS.md`。
- Claude Code desktop app / Claude Code CLI：最後可存成 `CLAUDE.md`。

你可以先對主 agent 說：

```text
請根據剛剛這次教材合作，幫我草擬一份工作區指令檔。
目標：讓下次 agent 修改教材前，能先遵守我的課程語言、修改範圍、個資紅線與 diff 回報習慣。
請先只產生草稿，不要寫入檔案。
```

主 agent 產生草稿後，老師確認內容，再請它存成 `AGENTS.md` 或 `CLAUDE.md`。草稿最小內容可以像這樣：

```text
# 我的教學工作守則

## 課程與語言
- 課程：請改成你的課名
- 語言：繁體中文

## 工作方式
- 修改前先說明會讀哪些檔案、會改哪些檔案。
- 動工前先反問我最多 3 個關鍵問題。
- 修改後請摘要 diff，列出需要老師人工確認的地方。

## 紅線
- 不處理學生姓名、學號、成績、聯絡方式。
- 不公開未授權教材或學生作品。
- 不要自行 push 或發布，除非我明確同意。
```

## H. 單次任務交辦模板

還沒有 `CLAUDE.md` 或 `AGENTS.md` 時，也可以直接用這個模板交辦。之後若同樣規矩反覆出現，再請主 agent 整理成工作區指令檔草稿。

```text
我正在準備一份教學資源頁。

目標：讓學生能在課後找到工具連結與第一步教學。
範圍：只包含本次課程提到的工具，不展開無關資源。
格式：請用 README.md 的結構回答。
限制：使用繁體中文；不要放學生個資；不要假造連結。
驗收標準：讀者 3 分鐘內能找到該下載或註冊哪個工具。

動工前，請先反問我最多 3 個需要確認的問題。
```

## I. 完成後要檢查

- 連結能不能打開。
- 是否不小心放入學生個資。
- Agent 是否產生了不存在的網址或功能。
- Git 是否已有修改前的存檔點。
- Agent 是否有先列計畫、再動手、再回報需要人工確認的地方。
- 如果這次合作有摩擦，是否已請主 agent 回顧哪些規矩下次要寫回 `CLAUDE.md` / `AGENTS.md`。
- 本場演講不需要完成 Skill、MCP、NotebookLM-py 或 subagent；那些是熟悉基本流程後的進階工具。
