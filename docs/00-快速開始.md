# 00 快速開始

這一頁給第一次參加工作坊的學員。目標不是一次學會所有工具，而是在 30 分鐘內完成三件事：

1. 找到課程資料可以放哪裡。
2. 建立一個可分享的 GitHub 資源頁。
3. 用一個 AI 工具完成有來源、有邊界的問答。

## A. 先準備帳號

| 帳號 | 用途 | 入口 |
|---|---|---|
| Google | NotebookLM、Gemini、Google Drive | <https://accounts.google.com/> |
| GitHub | 放 README、教材、課後資源 | <https://github.com/> |
| OpenAI | ChatGPT、Codex CLI | <https://chatgpt.com/> |
| Anthropic | Claude.ai、Claude Code | <https://claude.ai/> |

只想做零門檻練習的人，先有 Google 與 GitHub 就夠。

## B. NotebookLM 五分鐘練習

1. 打開 <https://notebooklm.google.com/>。
2. 建立一個新的 notebook。
3. 上傳一份可以分享的講義、簡報或 PDF。
4. 問：

```text
請用繁體中文整理這份教材的 5 個重點。
每一點都要附上來源引用。
```

5. 再問：

```text
請根據這份教材產生 5 題練習題。
不要使用教材以外的知識。
```

## C. GitHub README 十分鐘練習

1. 到 <https://github.com/new> 建立新 repo。
2. 勾選 `Add a README file`。
3. 在 README 寫三段：
   - 這是什麼課。
   - 學員應該先看哪份資料。
   - 有問題要去哪裡問。
4. 按 `Commit changes`。

這就是最小的課後資源頁。之後可以慢慢加入講義、連結、FAQ。

## D. AI 交辦模板

把下面這段貼到 ChatGPT、Claude.ai、Gemini 或 CLI agent：

```text
我正在準備一份教學資源頁。

目標：讓學生能在課後找到工具連結與第一步教學。
範圍：只包含本次課程提到的工具，不展開無關資源。
格式：請用 README.md 的結構回答。
限制：使用繁體中文；不要放學生個資；不要假造連結。
驗收標準：讀者 3 分鐘內能找到該下載或註冊哪個工具。

動工前，請先反問我最多 3 個需要確認的問題。
```

## E. 完成後要檢查

- 連結能不能打開。
- 是否不小心放入學生個資。
- AI 是否產生了不存在的網址或功能。
- README 第一屏是否看得出「這個 repo 是幹嘛的」。
